# Native Guard Roadmap v0.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement v0.3 from `docs/ROADMAP.md`: process icons, per-process termination, no-flicker main window refresh, and tray tooltip architecture labels.

**Architecture:** Keep testable list behavior in `NativeGuard.Core`, keep Windows icon extraction and process termination in `NativeGuard.App`, and update the WinUI collection by PID diff instead of clearing the list. The app continues to read system process data only; it does not sample CPU.

**Tech Stack:** .NET 10, C#, WinUI 3, MSTest, Win32 interop, Windows App SDK.

---

## File Structure

- `src/NativeGuard.Core/Processes/NonNativeProcessInfo.cs`: add executable path for icon lookup.
- `src/NativeGuard.Core/Processes/TrayTooltipFormatter.cs`: format tooltip rows as `name (architecture)`.
- `src/NativeGuard.Core/Processes/ProcessListDiff.cs`: immutable diff result for process list refresh.
- `src/NativeGuard.Core/Processes/ProcessListDiffer.cs`: calculate added, updated, removed, and sorted rows by PID.
- `tests/NativeGuard.Core.Tests/TrayTooltipFormatterTests.cs`: update tooltip expectations.
- `tests/NativeGuard.Core.Tests/ProcessListDifferTests.cs`: cover no-flicker diff behavior.
- `src/NativeGuard.App/Processes/Win32ProcessInfoProvider.cs`: populate executable path.
- `src/NativeGuard.App/Processes/ProcessIconProvider.cs`: extract and cache per-process icons.
- `src/NativeGuard.App/Processes/ProcessTerminator.cs`: terminate process by PID and return a displayable result.
- `src/NativeGuard.App/MainWindow.xaml`: show icon and terminate button.
- `src/NativeGuard.App/MainWindow.xaml.cs`: apply diff updates without clearing `Rows`.
