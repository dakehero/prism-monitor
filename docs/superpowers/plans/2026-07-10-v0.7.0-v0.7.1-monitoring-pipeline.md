# Prism Monitor v0.7.0-v0.7.1 Monitoring Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Prism Monitor's duplicate pull loops with one power-aware snapshot pipeline and add a retryable PID/creation-identity enrichment cache.

**Architecture:** `PrismMonitor.Core` supplies a platform-neutral coordinator, process-instance tracker, enrichment cache, cadence policy, and surface projections. `PrismMonitor.App` supplies Windows discovery/enrichment adapters and one `MonitoringHost` that schedules refreshes and distributes each successful snapshot to MainWindow, History, tray, and Toast consumers.

**Tech Stack:** .NET 10, C#, WinUI 3, Windows App SDK, narrow Win32 interop, MSTest.

## Global Constraints

- Hidden on battery uses a 30-second periodic fallback; tray interaction and power/visibility changes request an immediate refresh.
- AC and unknown power use 3 seconds; visible on battery uses 10 seconds.
- Use Windows/system process data only; do not add CPU sampling, ETW profiling, WMI watchers, elevation, or production dependencies.
- Preserve x86, x64, ARM64EC, and ARM64X classification, including PE fallback beyond `IsWow64Process2`.
- All runtime surfaces consume the same successful `MonitoringSnapshot`; a capture failure keeps the last-good snapshot.
- Process enrichment and enumeration must not run synchronously on the UI thread.
- Optional metadata failure must not remove an already-classified compatibility-mode process.
- Keep WinUI, tray, Toast, icons, process interop, power detection, and app lifetime in `PrismMonitor.App`.
- Keep testable process identity, enrichment state, rule projection, history behavior, and cadence policy in `PrismMonitor.Core`.
- Persist rules, settings, and history in app-local data; do not change packaging or signing.
- Use TDD for every production change and keep commits scoped.

```csharp
namespace PrismMonitor.Core.Power;

public static class RefreshSchedulePolicy
{
    private static readonly TimeSpan ResponsiveInterval = TimeSpan.FromSeconds(3);
    private static readonly TimeSpan VisibleBatteryInterval = TimeSpan.FromSeconds(10);
    private static readonly TimeSpan HiddenBatteryInterval = TimeSpan.FromSeconds(30);

    public static RefreshMode GetRefreshMode(PowerSource powerSource, bool isMainWindowVisible)
    {
        return RefreshMode.PeriodicBackground;
    }

    public static TimeSpan GetRefreshInterval(PowerSource powerSource, bool isMainWindowVisible)
    {
        if (powerSource != PowerSource.Battery)
        {
            return ResponsiveInterval;
        }

        return isMainWindowVisible ? VisibleBatteryInterval : HiddenBatteryInterval;
    }

    public static TimeSpan GetMainWindowRefreshInterval(PowerSource powerSource) =>
        GetRefreshInterval(powerSource, isMainWindowVisible: true);

    public static TimeSpan GetBackgroundRefreshInterval(PowerSource powerSource, bool isMainWindowVisible) =>
        GetRefreshInterval(powerSource, isMainWindowVisible);
}
```

`ProcessSnapshotTracker.Track` removes absent PIDs, preserves tracked state when creation identity matches, creates a new key when creation time/name changes, and assigns a first-observed key for missing creation time. Use `DistinctBy(ProcessId)` to reject duplicate observations deterministically.

```csharp
public sealed class ProcessSnapshotTracker
{
    private readonly Dictionary<int, TrackedProcess> _tracked = [];

    public IReadOnlyList<ProcessSnapshotInfo> Track(
        IReadOnlyList<ProcessObservation> observations,
        DateTimeOffset detectedAt)
    {
        HashSet<int> activeIds = observations.Select(item => item.ProcessId).ToHashSet();
        foreach (int exitedId in _tracked.Keys.Where(id => !activeIds.Contains(id)).ToList())
        {
            _tracked.Remove(exitedId);
        }

        List<ProcessSnapshotInfo> snapshots = [];
        foreach (ProcessObservation observation in observations.DistinctBy(item => item.ProcessId))
        {
            if (!_tracked.TryGetValue(observation.ProcessId, out TrackedProcess? tracked)
                || !tracked.Matches(observation))
            {
                tracked = new TrackedProcess(
                    observation.Name,
                    observation.CreationTime,
                    detectedAt,
                    new ProcessInstanceKey(
                        observation.ProcessId,
                        observation.CreationTime ?? detectedAt,
                        observation.CreationTime is not null));
                _tracked[observation.ProcessId] = tracked;
            }

            snapshots.Add(new ProcessSnapshotInfo(
                observation.ProcessId,
                observation.Name,
                observation.CpuTime,
                observation.CreationTime,
                tracked.DetectedAt,
                tracked.Key));
        }

        return snapshots;
    }

    private sealed record TrackedProcess(
        string Name,
        DateTimeOffset? CreationTime,
        DateTimeOffset DetectedAt,
        ProcessInstanceKey Key)
    {
        public bool Matches(ProcessObservation observation)
        {
            return CreationTime is not null || observation.CreationTime is not null
                ? CreationTime == observation.CreationTime
                : string.Equals(Name, observation.Name, StringComparison.OrdinalIgnoreCase);
        }
    }
}
```
